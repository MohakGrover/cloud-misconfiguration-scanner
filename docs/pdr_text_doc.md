# Documentation: `pdr_text.txt`

## Overview
`pdr_text.txt` contains the extracted text of the **Product Design & Requirements (PDR)** document for the "Multi-Cloud Misconfiguration Security Scanner". This file serves as the raw text reference for the project's vision, requirements, and architecture.

## Document Summary
The document outlines the plan to build a lightweight, education-focused **Cloud Security Posture Management (CSPM)** tool.

### Key Sections

1.  **Vision & Problem**:
    - **Goal**: Empower security learners to detect and understand cloud misconfigurations across AWS, Azure, and GCP.
    - **Problem**: Existing tools are too expensive or complex for learners.

2.  **Target Users**:
    - **"Sam the Security Learner"**: Student/Engineer learning cloud security.
    - **"Alex the Small Team Lead"**: DevOps lead needing quick security visibility.

3.  **Core User Journeys**:
    - First-time setup and scan.
    - Scanning Infrastructure-as-Code (IaC) like Terraform.
    - Learning via a "Validation Lab" (intentionally misconfigured environment).
    - Compliance reporting (CIS Benchmarks).

4.  **Functional Requirements (FR)**:
    - **FR-1**: Cloud Config Collection (AWS/Azure/GCP).
    - **FR-2**: IaC Scanning (Terraform parsing).
    - **FR-3**: Modular Rules Engine (YAML/JSON based).
    - **FR-4**: Misconfiguration Detection (Storage, Network, IAM, Database, Compute).
    - **FR-5**: Validation Lab (Seeded misconfigurations, `lab_nuke.py` for cleanup).
    - **FR-6**: Web Dashboard (Findings, filters, remediation).

5.  **Technical Stack (MVP)**:
    - **Backend**: Python 3.9+, boto3.
    - **Database**: DuckDB.
    - **Frontend**: React/Vue.js + Tailwind CSS.
    - **IaC Parsing**: python-hcl2 or Checkov integration.

6.  **Architecture**:
    - **CLI Tool** & **Web Dashboard** -> **Orchestration Layer** -> **Collectors** & **Rules Engine** -> **Storage**.
